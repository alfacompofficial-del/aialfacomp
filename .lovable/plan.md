
# Nexus: десктоп + мобильная сборка + публичное API

## Что получится в итоге

1. **Веб-приложение Nexus** (уже работает) — добавим страницу "Скачать"
2. **Файл `Nexus-Setup.exe`** для Windows — установщик десктоп-приложения
3. **Файл `Nexus.apk`** для Android — устанавливается напрямую на телефон
4. **Раздел "Для разработчиков"** на сайте — регистрация → получение API-ключа `nx-xxxxx`
5. **Публичный API** на `apiai.alfacomp.uz` — `/v1/chat` с поддержкой инструментов Nexus и стриминга
6. **Лимиты** — 100 запросов в день на бесплатный ключ (можем поменять цифру)

---

## Этап 1 — Десктоп-сборка (.exe)

Используем **Electron + @electron/packager**:
- Создаём `electron/main.cjs` — окно, которое грузит ваш сайт `https://aialfacomp.lovable.app`
- Меняем `vite.config.ts`: `base: './'` (для file:// загрузки оффлайн-версии в будущем)
- Скрипт сборки → `Nexus-Setup-win.zip` (внутри `Nexus.exe` + ресурсы)
- Кладём готовый файл в Lovable Cloud Storage в публичный бакет `downloads`
- На сайте — кнопка "Скачать для Windows" с прямой ссылкой

**Важно:** Песочница Lovable собирает только **Linux/Windows zip-сборку**. Полноценный `.exe`-инсталлятор (NSIS) собирается только на Windows-машине. Что я могу сделать здесь:
- Собрать `Nexus-win-x64.zip` — пользователь распаковывает и запускает `Nexus.exe`
- Это работает на 100% Windows-машин, не требует установки

Если нужен именно установщик с мастером "Далее-Далее-Готово" — это отдельная история, делается на вашем ПК через Electron Builder.

## Этап 2 — Android-сборка (.apk)

Используем **Capacitor**:
- `npx cap init` с `appId: uz.alfacomp.nexus`, `appName: Nexus`
- Добавляем Android-платформу
- Собрать готовый `.apk` **в песочнице нельзя** — для этого нужен Android Studio + Android SDK (~10 ГБ), которых здесь нет

**Что я сделаю реально:**
- Полностью настрою проект Capacitor (манифест, иконки, splash screen, конфиг)
- Подготовлю инструкцию из 4 шагов для сборки `.apk` на вашем ПК (бесплатно, через Android Studio)
- Альтернатива: подключить **GitHub Actions** — при экспорте в GitHub он сам соберёт `.apk` и положит в релиз. Тогда вы вообще ничего не устанавливаете, только нажимаете "скачать релиз" на GitHub

Скажите после плана, какой путь Android выбираем.

## Этап 3 — Страница "Скачать" в приложении

- Маршрут `/download` с двумя большими карточками: Windows и Android
- Кнопки качают файлы из бакета `downloads` через fetch+blob (как в инструкции для Chrome-расширений)
- Версия, размер файла, дата сборки
- Инструкция установки (распаковать zip / разрешить установку из неизвестных источников)

## Этап 4 — Публичный API на apiai.alfacomp.uz

### База данных
Новые таблицы:
- `api_keys` — `id, user_id, key_hash (sha-256), key_prefix (nx_xxx для отображения), name, created_at, last_used_at, revoked`
- `api_usage` — `id, key_id, endpoint, tokens_in, tokens_out, created_at` (для подсчёта лимита)

RLS: пользователь видит только свои ключи и свою статистику.

### Раздел в приложении `/developers`
- Список ключей пользователя
- Кнопка "Создать новый ключ" → показывается **один раз** полный ключ `nx_live_abc123...` (потом только префикс)
- Кнопка "Отозвать"
- График использования за 7/30 дней
- Документация: примеры curl, Python, JS

### Edge Function `public-api`
- Принимает `Authorization: Bearer nx_live_xxx`
- Хеширует ключ → ищет в `api_keys` → проверяет `revoked = false`
- Считает использование за последние 24ч → если >100, возвращает 429
- Прокидывает запрос в существующую логику `chat` (со всеми инструментами Nexus)
- Поддерживает streaming SSE как обычный чат
- Логирует в `api_usage`

### Поддомен apiai.alfacomp.uz
Поддомены настраиваются в **Project Settings → Domains** в Lovable. Я подскажу какие DNS-записи добавить у регистратора `alfacomp.uz`:
- A-запись `apiai` → `185.158.133.1`
- TXT-запись `_lovable` для верификации

После подключения API будет доступен как `https://apiai.alfacomp.uz/v1/chat`.

**Важно:** в Lovable один проект = один домен. Поэтому варианта два:
- **Вариант A (просто):** API доступен на том же `aialfacomp.lovable.app/v1/chat`. Поддомен `apiai.alfacomp.uz` подключаем как **алиас** к этому же проекту — будет работать оба адреса
- **Вариант B (по-взрослому):** отдельный Lovable-проект только под API. Но это дублирование — не рекомендую

Идём по варианту A.

## Технический раздел

**Стек добавлений:**
- `electron`, `@electron/packager` — десктоп
- `@capacitor/core`, `@capacitor/cli`, `@capacitor/android` — мобайл
- Storage bucket `downloads` (public) для .zip и .apk
- 2 новые таблицы + RLS
- 1 новая edge function `public-api`
- 4 новых страницы: `/download`, `/developers`, `/developers/keys`, `/developers/docs`

**Безопасность:**
- API-ключи хранятся **только как SHA-256 хеш** — даже мы не видим оригинал
- Полный ключ показывается пользователю один раз при создании
- Rate limiting на уровне БД (счётчик за 24ч)
- Префикс `nx_live_` для production, потом сможем добавить `nx_test_`

**Что попрошу у вас по ходу:**
1. Когда дойдём до домена — добавить DNS-записи у регистратора `alfacomp.uz` (3 минуты)
2. Для сборки `.apk` — решить: соберёте сами в Android Studio или подключаем GitHub Actions

## План работ

```text
Этап 1 — Подготовка
  ├─ Storage bucket "downloads" (public)
  ├─ Таблицы api_keys, api_usage + RLS
  └─ Страница /download (UI готов, файлы добавим после сборки)

Этап 2 — Десктоп сборка
  ├─ Electron конфиг (main.cjs, vite base)
  ├─ Сборка Nexus-win-x64.zip
  └─ Загрузка в bucket downloads

Этап 3 — Android подготовка
  ├─ Capacitor init + android платформа
  ├─ Иконки, splash, манифест
  └─ Инструкция/GitHub Action на ваш выбор

Этап 4 — Public API
  ├─ Edge function public-api с авторизацией по ключу
  ├─ Rate limiting (100/день)
  ├─ Стриминг + инструменты Nexus
  └─ Логирование в api_usage

Этап 5 — UI разработчика
  ├─ /developers — лендинг "Получи API ключ"
  ├─ /developers/keys — управление ключами
  ├─ /developers/docs — документация с примерами
  └─ График использования

Этап 6 — Домен
  ├─ Подключение apiai.alfacomp.uz
  └─ Тест публичного API
```

Готов начинать с Этапа 1, как только одобрите.
