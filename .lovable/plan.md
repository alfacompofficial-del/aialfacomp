# План: ускорение Nexus + режим /think + реальные инструменты

## Проблемы которые сейчас есть

1. **Медленно**: edge-функция `chat/index.ts` на каждой итерации tool-loop делает **два запроса к AI** — сначала non-stream «думает», потом ещё один stream запрос с тем же контекстом. Это удваивает задержку и токены.
2. **Не может проверять сайты/GitHub**:
  - `web_search` через DuckDuckGo HTML — нестабилен, часто блокируется ботом, парсинг ломается.
  - `github_search` без `GITHUB_TOKEN` упирается в лимит 10 req/min.
  - Нет инструмента «открыть конкретный URL» — модель не может прочитать страницу.
  - `fetch_docs` ничего не делает — просто возвращает заглушку.
3. **Нет /think**: нельзя попросить модель думать дольше и точнее.

## Что сделаю

### 1. Ускорение edge-функции `supabase/functions/chat/index.ts`

- Убрать двойной запрос. Делаем стриминг **сразу с инструментами** — `stream: true` + `tools: TOOLS` поддерживается gateway. Из стрима парсим `tool_calls` дельты, накапливаем их, выполняем, и продолжаем стримить дальше — без второго полного non-stream вызова.
- Если в первой итерации модель не вызывает инструменты — стрим уходит к клиенту напрямую с первого запроса (а не со второго). Это **в 2 раза быстрее** для обычных вопросов.
- Поднять лимит итераций tool-loop с 4 до 6 (для /think).

### 2. Команда `/think` (режим глубокого размышления)

Frontend (`src/pages/Chat.tsx`):

- В `onKeyDown` распознаём префикс `/think` (как уже делается с `/image`).
- Если найден — убираем префикс из текста, выставляем флаг `think: true` в теле запроса, плюс показываем бейдж «🧠 Глубокое размышление» рядом с pending-сообщением.
- Добавить кнопку 🧠 (Brain) в composer рядом с 🪄 — нажатие переключает режим на следующее сообщение.

Backend (`chat/index.ts`):

- Принимаем `think: boolean` в body.
- Если `think === true`:
  - Используем модель `openai/gpt-5` (сильнее в reasoning) вместо дефолтной `gemini-3-flash-preview`.
  - Добавляем в payload `reasoning: { effort: "high" }`.
  - В системный промпт добавляем блок: «Сейчас режим глубокого размышления. Сначала проанализируй задачу пошагово, проверь факты через инструменты (web_search/fetch_url/github_search), и только затем дай точный, обоснованный ответ со ссылками на источники.»
  - Принудительно требуем хотя бы один tool-call если запрос фактологический (через system prompt).

### 3. Реально рабочие инструменты

Заменю/добавлю в `chat/index.ts`:

**a) `web_search` — переписать на Perplexity API** (если есть `PERPLEXITY_API_KEY`):

- `sonar` модель, возвращает ответ + citations.
- Fallback: текущий DuckDuckGo, если ключа нет.
- **Понадобится**: добавить секрет `PERPLEXITY_API_KEY` через secrets-инструмент. Спрошу у пользователя, готов ли подключить (если нет — оставлю улучшенный DuckDuckGo с user-agent ротацией).

**b) Новый инструмент `fetch_url**` — открыть и прочитать любой URL:

- `fetch(url)`, чистка HTML → текст (убрать `<script>/<style>`, оставить текст), обрезка до 8000 символов.
- Это закрывает «не может проверить сайты».

**c) `github_search` — улучшить**:

- Если есть `GITHUB_TOKEN` — используем (5000 req/h).
- Добавить поиск по коду (`/search/code`), не только по репозиториям.
- Для конкретного репо/файла — комбинировать с `fetch_url` (raw.githubusercontent.com).

**d) `fetch_docs` — сделать рабочим**:

- Маппинг популярных библиотек на их docs URL (react.dev, nextjs.org/docs, tailwindcss.com/docs, и т.д.) → внутри вызывает `fetch_url`.

### 4. UI индикация инструментов

В `MessageBubble` (уже есть `toolsUsed`) — показывать названия использованных инструментов ("🔍 web_search · 📄 fetch_url"), не только счётчик. Backend уже шлёт `meta`-event, расширим: `meta: { tools: ["web_search", "fetch_url"] }`.

## Технические детали

**Изменяемые файлы:**

- `supabase/functions/chat/index.ts` — переписать tool-loop на streaming-first, добавить `fetch_url`, переписать `web_search`, маппинг `fetch_docs`, поддержка `think` и `reasoning`.
- `supabase/functions/_shared/nexus-agent.ts` — синхронизировать те же изменения (используется в `public-api`).
- `src/pages/Chat.tsx` — `/think` команда, кнопка 🧠, бейдж режима, передача `think` в body, отображение списка инструментов.

**Секреты:**

- Спрошу про `PERPLEXITY_API_KEY` (опционально, для качественного web-поиска).
- `GITHUB_TOKEN` (опционально, чтобы не упираться в лимит).

**Ожидаемый эффект:**

- Обычный ответ — в 1.5–2× медленее  но точнее (один запрос вместо двух).
- `/think` — медленнее, но заметно точнее, со ссылками на источники. и должен лучше писать код и лучше отвечать на вопросы 
- Реально умеет открывать сайты и GitHub.
