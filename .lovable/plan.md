
# Свой ИИ-агент с инструментами и памятью

## Что это будет

Собственный ИИ-агент (назовём его временно **Nexus** — финальное имя выберете вы) с уникальной личностью и набором инструментов. Под капотом — Lovable AI Gateway (Gemini 3 Flash по умолчанию для скорости + Gemini 2.5 Pro для сложных задач), но логика, инструменты, память, фильтры и интерфейс — полностью наши.

Это и есть "свой ИИ-агент" в современном понимании (так устроены Cursor, Perplexity, Windsurf).

## Основные возможности

### 1. Аутентификация
- Регистрация по email + паролю **без подтверждения почты** (мгновенный вход)
- Google Sign-In как альтернатива
- Защита всех данных через RLS — каждый пользователь видит только свои чаты

### 2. Чат-интерфейс (Midnight Indigo стиль)
- Боковая панель со списком чатов (история всех бесед)
- Создание/удаление/переименование чатов
- Streaming-ответы (текст появляется по мере генерации)
- Markdown-рендеринг с подсветкой синтаксиса для кода
- Кнопка "Копировать" на каждом блоке кода
- **Кнопка "Протестировать"** на блоках HTML/CSS/JS → открывает встроенный iframe-превью прямо в чате
- Поле ввода с авто-ресайзом, поддержкой Shift+Enter
- Индикатор "агент думает..." и какой инструмент сейчас использует

### 3. Загрузка файлов и фото
- Drag & drop или кнопка прикрепления
- Поддержка: изображения (JPG, PNG, WebP), PDF, текстовые файлы, код
- Изображения анализируются мультимодально (агент "видит" фото)
- PDF/DOCX парсятся на текст и передаются в контекст
- Файлы хранятся в Lovable Cloud Storage (приватные бакеты на пользователя)

### 4. Инструменты агента (tool calling)
Агент сам решает, когда использовать инструмент:

| Инструмент | Когда вызывается | Источник |
|---|---|---|
| `web_search` | Текущие события, новости, факты с цитатами | Perplexity API |
| `github_search` | Поиск примеров кода, репозиториев | GitHub API (публичный) |
| `wikipedia_lookup` | Энциклопедические факты, даты, биографии | Wikipedia API (бесплатный) |
| `fetch_docs` | Документация фреймворков (React, Next.js, Python и т.д.) | Прямой fetch с context7/официальных источников |
| `analyze_image` | Описание/OCR загруженного фото | Встроено в Gemini |
| `analyze_document` | Извлечение текста из PDF/DOCX | Backend парсер |

### 5. Защита и фильтры (Guardrails)
- Системный промпт запрещает мат, оскорбления, помощь в нарушении законов
- Серверная пре-фильтрация запросов (классификатор намерений на лёгкой модели)
- Пост-фильтрация ответов: если модель что-то пропустила — блокируем
- Rate limiting: лимит запросов в минуту на пользователя (защита от абуза)
- Все запрещённые темы логируются для последующего анализа

### 6. Личность агента
- Уникальный системный промпт: дружелюбный, профессиональный, говорит на русском по умолчанию
- Помнит контекст всей беседы (полная история сообщений)
- Представляется как **ваш** агент, а не как Gemini/GPT

## Технический раздел

**Стек:**
- Frontend: React + Tailwind, react-markdown + rehype-highlight для кода, react-syntax-highlighter
- Backend: Lovable Cloud (Supabase) — auth, БД, storage, edge functions
- AI: Lovable AI Gateway (gemini-3-flash-preview по умолчанию, переключение на gemini-2.5-pro для сложных задач)
- Streaming: SSE из edge function в браузер

**База данных:**
- `profiles` — данные пользователя (display_name, avatar)
- `conversations` — чаты (id, user_id, title, created_at)
- `messages` — сообщения (id, conversation_id, role, content, attachments jsonb, tool_calls jsonb)
- `attachments` storage bucket — файлы пользователей (приватный, RLS)
- Все таблицы с RLS: пользователь видит только свои данные

**Edge Functions:**
- `chat` — основной агентский цикл со streaming, tool calling, guardrails
- `web-search` — Perplexity proxy (требует ключ Perplexity)
- `github-search` — GitHub API proxy (публичный, без ключа для базового поиска)
- `parse-document` — извлечение текста из PDF/DOCX
- `moderate` — классификатор запрещённого контента

**Секреты, которые понадобятся (запрошу после старта):**
- `PERPLEXITY_API_KEY` — для веб-поиска (получить на perplexity.ai → API)
- (Опционально) `GITHUB_TOKEN` — увеличит лимиты GitHub API
- `LOVABLE_API_KEY` — настроится автоматически

**MCP документация:** Полноценное MCP-подключение из браузера сложно (MCP-серверы обычно локальные). Сделаем альтернативу: прямой fetch к официальной документации популярных библиотек через специальный инструмент `fetch_docs` — функционально это даст тот же результат.

## План реализации (поэтапно)

```text
Этап 1 — Фундамент
  ├─ Lovable Cloud + auth (email/пароль без подтверждения, Google)
  ├─ Таблицы: profiles, conversations, messages + RLS
  ├─ Storage bucket для вложений
  └─ Базовый layout: sidebar + chat area, тема Midnight Indigo

Этап 2 — Чат с ИИ (MVP)
  ├─ Edge function `chat` со streaming
  ├─ Системный промпт с личностью + guardrails
  ├─ Markdown рендеринг + подсветка кода + кнопка копировать
  ├─ Создание/выбор/удаление чатов, история
  └─ Загрузка изображений (мультимодальный анализ)

Этап 3 — Инструменты агента
  ├─ Tool calling в edge function
  ├─ Web search (Perplexity)
  ├─ GitHub search
  ├─ Wikipedia
  ├─ Fetch docs
  └─ UI индикаторы "агент использует X"

Этап 4 — Файлы и HTML песочница
  ├─ Загрузка PDF/DOCX + парсинг
  ├─ Кнопка "Протестировать" на HTML блоках → iframe sandbox в чате
  └─ Превью с возможностью развернуть на весь экран

Этап 5 — Полировка
  ├─ Rate limiting
  ├─ Дополнительная пост-модерация ответов
  ├─ Переключатель моделей (быстрая/умная)
  └─ Анимации, мобильная адаптация
```
